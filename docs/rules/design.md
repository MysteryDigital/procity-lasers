# DESIGN RULES — MANDATORY

Applies when: any UI, layout, component, CSS, or styling work.

Violations are bugs, not style preferences.

## Palette

- **Neutrals only as base:** charcoal, slate, off-white. No other base colours.
- **One accent colour** — technical blue/teal direction. Use sparingly: one active
  state, one CTA, one highlight. Never two competing accents on the same page.
- Photography carries all other colour. The UI must not compete with it.
- All colour values must come from design tokens. No hardcoded hex/rgb values in
  components or CSS.

## Typography

- **DM Sans** is the confirmed typeface — use it for all UI text.
- No decorative or display faces.
- A restrained serif for large headings is permitted as an optional heritage nod
  only — confirm before use.
- Clear hierarchy: generous sizing, ample line-height. Readers skim.
- No more than two typefaces total.

## Spacing & layout

- Grid-driven and spacious. Whitespace signals quality — never fill space for its
  own sake.
- Hairline rules and crisp dividers as structural elements. No thick decorative
  borders.
- Deliberate alignment. Nothing visually loose.

## Icons & graphics

- Thin-line icon set only. Consistent stroke weight across all icons.
- No decorative illustration or pattern. Photography is the visual content.
- The logo is a drop-in asset — do not derive colours, shapes, or motifs from it.

## Photography

- Real project imagery only. No generic stock where real work can be shown.
- Before/after is a core UI pattern (slider), not just a gallery treatment.
- Images must never be cropped or styled in a way that obscures the result evidence.
